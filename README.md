# Web App with Nginx and Docker Compose

Простое веб-приложение (Python HTTP Server), доступное через обратный прокси (Nginx), запущенное в Docker.

## Стек технологий

- **Docker & Docker Compose**: Контейнеризация и оркестрация
- **Nginx (alpine)**: Обратный прокси-сервер (Reverse Proxy)
- **Python (alpine)**: Backend-приложение (встроенный `http.server`)

## Запуск проекта

Убедитесь, что у вас установлены `docker` и `docker-compose` (или плагин `docker compose`).

Выполните сборку и запуск приложения в фоновом режиме:

```bash
docker-compose up -d --build
```

## Проверка работоспособности

Вы можете сделать запрос на локальный хост (по умолчанию порт 80):

```bash
curl http://localhost
```

Ожидаемый ответ:

```
Hello from Effective Mobile!
```

Также вы можете проверить статус контейнеров (в проекте настроены `healthcheck`):

```bash
docker-compose ps
```

## Как работает архитектура

Взаимодействие происходит внутри изолированной docker-сети `em_network`.

```text
                  +-----------------------------------+
                  |           em_network              |
                  |                                   |
[ User / curl ]   |   +----------+      +---------+   |
   (Host)  -----------| em_nginx |----->| backend |   |
(http://localhost)|   | (:80)    |      | (:8080) |   |
                  |   +----------+      +---------+   |
                  |                                   |
                  +-----------------------------------+
```

1.  Пользователь обращается к порту `80` на хост-машине.
2.  Этот порт проброшен **только** на контейнер `nginx` (`em_nginx`).
3.  Nginx принимает HTTP-запрос и выступает в роли reverse-proxy. Согласно конфигурации `nginx.conf`, он проксирует запрос по имени сервиса (`backend`) на порт `8080`.
4.  Backend-сервер (`em_backend`) — это минимальный Python-сервер, который работает от непривилегированного пользователя. Он слушает порт `8080` внутри сети Docker.
5.  Backend-сервер отвечает текстом `"Hello from Effective Mobile!"`, и Nginx передает этот ответ обратно пользователю.
6.  Порт `8080` backend-сервера не опубликован на хост-машине, поэтому прямой доступ к backend невозможен в обход Nginx.

## Безопасность и лучшие практики (реализовано)

- **Минимальные базовые образы:** Используются образы `alpine` для меньшего размера и уменьшения поверхности атаки.
- **Непривилегированный запуск:** Backend приложение выполняется от непривилегированного пользователя (`appuser`), а не от root.
- **Изоляция сети:** Backend не пробрасывает порты наружу. Связь между Nginx и приложением осуществляется только внутри внутренней Docker-сети `em_network`.
- **Healthchecks:** Настроены healthcheck для Nginx и Backend, что позволяет Docker контролировать доступность компонентов.
- **Скрытие версии Nginx:** Используется директива `server_tokens off;` для защиты от автоматизированного сканирования версий.
